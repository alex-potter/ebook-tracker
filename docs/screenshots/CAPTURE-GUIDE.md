# Screenshot Capture Guide

Instructions for capturing screenshots to fill the README placeholders.

## Tools

- **Screenshots:** ShareX (Windows), Flameshot (Linux), Cmd+Shift+4 (macOS)
- **GIFs:** ScreenToGif (Windows), LICEcap, Kap (macOS)

## Settings

- Browser window: **1280x800**
- Use **dark mode**
- Load a **public-domain book** (e.g., from Project Gutenberg)
- Analyze at least 3-4 chapters so dashboards have meaningful data

## Required Captures

| File | What to show |
|------|-------------|
| `hero.png` | Full character dashboard, 3-4 cards visible, chapter sidebar open |
| `characters.png` | Characters tab with cards showing varied statuses (alive, dead, unknown) |
| `locations.png` | Locations tab with several location entries |
| `map.png` | Map tab with pinned locations visible |
| `arcs.png` | Arcs tab with 2+ narrative arcs tracked |

## Optional Captures

| File | What to show |
|------|-------------|
| `upload.png` | Upload zone with welcome banner visible |
| `chat.png` | Chat panel with a spoiler-free Q&A exchange |
| `settings.png` | Settings modal showing AI provider options |
| `mobile.png` | Mobile viewport (375px wide) showing the dashboard |
| `analyze.gif` | Short GIF of the analyze flow: pick chapter, click Analyze, results appear |
